# Deploy to Render

## What you need
- A [GitHub](https://github.com) account
- A [Render](https://render.com) account (free, sign in with GitHub)

---

## Step 1 — Push to GitHub

In your `hostel_app` folder:

```bash
git init
git add .
git commit -m "Initial commit"
```

Go to [github.com/new](https://github.com/new) → create a new repo → then:

```bash
git remote add origin https://github.com/YOUR_USERNAME/hostel-app.git
git branch -M main
git push -u origin main
```

---

## Step 2 — Create Render Web Service

1. Go to [render.com](https://render.com) → **New → Web Service**
2. Connect your GitHub repo
3. Render detects `render.yaml` automatically — confirm:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app --bind 0.0.0.0:$PORT --workers 1 --timeout 120`

---

## Step 3 — Set Environment Variables

In Render → your service → **Environment** tab → add:

| Key | Value |
|-----|-------|
| `ADMIN_ID` | your chosen ID (not 1234) |
| `ADMIN_PASS` | your strong password (not 5005) |
| `DB_PATH` | `/data/hostel.db` |

`SECRET_KEY` is auto-generated by `render.yaml`.

**Important — no quotes around values.** Just:
```
ADMIN_ID=myid
ADMIN_PASS=mypassword
```

---

## Step 4 — Add Persistent Disk (keeps your database)

Render free tier resets files on every deploy.
To keep your database permanently:

1. Render dashboard → your service → **Disks** tab
2. **Add Disk** → Name: `hostel-data` → Mount Path: `/data` → Size: 1 GB
3. This costs **$1/month** on Render

> **Free alternative:** If you don't add a disk, the app still works —
> but data resets on every deploy. Fine for testing, not for real use.

---

## Step 5 — Deploy

Click **Deploy** → wait ~2 minutes → your app is live at:
`https://hostel-assembly.onrender.com`

---

## Updating later

```bash
git add .
git commit -m "your change"
git push
```

Render auto-redeploys on every push.

---

## Local development

```bash
cp .env.example .env
# Edit .env with your values (no quotes)

pip install -r requirements.txt
python app.py
# Open http://localhost:5000
```

---

## .env file format — important

✅ Correct:
```
ADMIN_ID=myid
ADMIN_PASS=mypassword
SECRET_KEY=someLongRandomString
DB_PATH=hostel.db
```

❌ Wrong (quotes cause issues):
```
ADMIN_ID="myid"
ADMIN_PASS="mypassword"
```
